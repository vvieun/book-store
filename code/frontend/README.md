# React Frontend для системы рекомендаций

Веб-интерфейс для взаимодействия с API системы рекомендаций.

## Возможности

### 📦 Продукты
- Просмотр списка продуктов
- Фильтрация по категориям
- Просмотр популярных продуктов
- Детальная информация о продукте

### ⭐ Оценки
- Создание оценок продуктов
- Интерактивный выбор рейтинга (звёздочки)
- Просмотр истории оценок пользователя

### ✨ Рекомендации
- Просмотр персонализированных рекомендаций
- Расчёт новых рекомендаций
- Показ релевантности рекомендаций

### 📊 Статистика
- Общая статистика пользователя
- Количество оценок, просмотров, покупок
- История взаимодействий

### 🏥 Мониторинг
- Проверка состояния API
- Статистика Redis
- Ссылки на документацию

## Запуск

### С Docker (рекомендуется)

```bash
cd ..  # В корень проекта
./run.sh start
```

Frontend будет доступен по адресу: http://localhost:3000

### Локально (без Docker)

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm start

# Сборка для production
npm build
```

## API Integration

Приложение взаимодействует с Spring Boot API через следующие endpoints:

### Книги
- `GET /api/books/{id}` — получить книгу
- `GET /api/books/category/{id}` — книги категории
- `GET /api/books/top` — топ по рейтингу

### Отзывы/оценки
- `POST /api/reviews` — создать/обновить отзыв
- `GET /api/reviews/user/{id}` — отзывы пользователя
- `GET /api/reviews/book/{id}` — отзывы по книге

### Рекомендации
- `GET /users/{id}/recommendations` — получить рекомендации
- `POST /users/{id}/recommendations/calculate` — рассчитать рекомендации

### Взаимодействия
- `POST /interactions` — зарегистрировать действие (view, cart, purchase)

### Статистика
- `GET /users/{id}/statistics` — статистика пользователя
- `GET /health` — состояние системы

## Конфигурация

### Переменные окружения

Создайте файл `.env` в корне `frontend/`:

```env
REACT_APP_API_URL=http://localhost:8000
```

### Прокси (для локальной разработки)

В `package.json` настроен прокси на API:

```json
"proxy": "http://api:8000"
```

Это работает только внутри Docker. Для локальной разработки используйте `REACT_APP_API_URL`.

## Компоненты

### ProductList
Отображает список продуктов с фильтрацией по категориям или популярности.

**Возможности:**
- Переключение между режимами (категория/популярные)
- Выбор категории
- Отображение рейтинга и цены
- Переход к детальной странице

### ProductDetail
Детальная информация о продукте с возможностью взаимодействия.

**Возможности:**
- Автоматическая регистрация просмотра
- Добавление в корзину
- Покупка
- Переход к созданию оценки

### Recommendations
Персонализированные рекомендации пользователя.

**Возможности:**
- Просмотр рекомендаций
- Пересчёт рекомендаций
- Отображение релевантности (score)

### CreateRating
Форма создания оценки продукта.

**Возможности:**
- Ввод ID продукта
- Интерактивный выбор рейтинга (звёздочки)
- Автозаполнение ID при переходе со страницы продукта

### UserStatistics
Статистика активности пользователя.

**Возможности:**
- Общие метрики (оценки, просмотры, покупки)
- История оценок
- Визуализация данных

### ApiStatus
Мониторинг состояния системы.

**Возможности:**
- Проверка доступности API
- Статус PostgreSQL и Redis
- Автообновление каждые 10 секунд
- Ссылки на документацию

## Разработка

### Добавление нового компонента

1. Создайте файл в `src/components/`:
```javascript
// NewComponent.js
import React from 'react';
import './NewComponent.css';

function NewComponent() {
  return <div>Новый компонент</div>;
}

export default NewComponent;
```

2. Создайте стили `NewComponent.css`

3. Добавьте маршрут в `App.js`:
```javascript
import NewComponent from './components/NewComponent';

<Route path="/new" element={<NewComponent />} />
```

### Добавление API метода

В `src/api.js`:
```javascript
export const newMethod = (param) => 
  api.get(`/new-endpoint/${param}`);
```

## Troubleshooting

### API недоступен

Убедитесь, что API (Java) запущен:
```bash
curl http://localhost:8000/health
```

### Ошибки CORS

Проверьте, что в api-java включён CORS для `http://localhost:3000`.

### Не загружаются данные

1. Откройте Developer Tools (F12)
2. Проверьте вкладку Network
3. Посмотрите на ошибки запросов
4. Проверьте состояние API: `/status`

### Проблемы с Docker

```bash
# Пересоздать контейнер
docker-compose up --build frontend

# Посмотреть логи
docker-compose logs frontend
```
